# Spending.gov.ua та НАЗК (Державні реєстри)

Цей документ описує автоматичну інтеграцію з порталом публічних коштів (Spending.gov.ua) та реєстром декларацій (НАЗК).

## Spending.gov.ua (Публічні кошти)
- **Ендпоінт:** `https://api.spending.gov.ua/api/v2/api/transactions`
- **Тип завантаження:** Incremental REST (по днях)
- **Планувальник:** Запускається щодобово для витягування транзакцій за вчорашній день.
- **Сутності:** 
  - `Transaction` (Сума, Призначення, Дата)
  - `Company` (Платник - payer)
  - `Company` (Одержувач - receiver)
- **Графи (Neo4j):** Створюється зв'язок `(Company)-[:PAID]->(Company)`.

## НАЗК (Декларації)
- **Ендпоінти:** 
  - Список: `https://public-api.nazk.gov.ua/v2/documents/list`
  - Деталі: `https://public-api.nazk.gov.ua/v2/documents/{id}`
- **Тип завантаження:** Incremental REST (пагінація, фільтр від дати)
- **Планувальник:** Запускається кожні 6 годин.
- **Сутності:** 
  - `Declaration`
  - `PEP` (Політично значуща особа)
  - `Asset` (Активи, нерухомість)
- **Векторизація (Qdrant):** Текст декларації, ім'я та посада передаються в `searchable_text` для подальшої побудови семантичного індексу (embeddings).
