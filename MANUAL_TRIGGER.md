# Інструкція запуску ETL процедури (Manual ETL Trigger)

Оскільки автоматичний запуск заблоковано налаштуваннями безпеки («Operation not permitted» при доступі до мережі та файлів), необхідно виконати запуск вручну. Ваша система вже налаштована на потокову обробку.

### 1. Переконайтеся, що Backend запущено
Якщо він ще не запущений, виконайте в терміналі:
```bash
# В папці services/api-gateway
./.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8090
```

### 2. Запуск ETL процесу
Використовуйте цю команду для запуску ETL для вашого файлу.
**Важливо:** Переконайтеся, що файл фізично доступний серверу (скопіюйте його в папку проекту, якщо він на робочому столі).

**Команда:**
```bash
curl -X POST "http://localhost:8090/v25/etl/process-local?file_path=/app/scripts/customs.xlsx&dataset_type=customs_march_2024"
```
*(Замініть `/app/scripts/customs.xlsx` на реальний шлях до файлу всередині Docker контейнера або локальної папки)*.

### 3. Спостереження (Web UI)
1. Відкрийте панель управління: `http://localhost:3001`
2. Перейдіть на вкладку **"Evolution & ETL"** (або "Ingestion").
3. Спостерігайте за секцією **"Sovereign ETL Registry"**.
4. Ви побачите, як статус просувається від **"Processing..."** до показу кількості записів (наприклад, "2.5k mapped", "10.2k indexed") в реальному часі.
