# 🚀 PREDATOR ELITE: Мастер-Блупринт та Стратегія Відновлення (Disaster Recovery)

Цей документ містить повну концепцію, архітектуру та інструкції з відновлення Агента Predator. У випадку критичного збою (зламався VS Code, знищено сервер NVIDIA або втрачено конфіги), ви можете відтворити всю екосистему, слідуючи цьому посібнику.

---

## 🏗 Архітектура Системи

Система Predator складається з наступних ключових компонентів:
1. **VS Code (MacBook):** Головний мозок (Agent Predator). Використовує розширення Continue з пропрієтарним `config.yaml`, що містить правила поведінки (українська мова, автономність, заборона запитів на дозвіл) та підключені MCP сервери (GitHub, Postgres, Puppeteer).
2. **NVIDIA Compute Node:** Обчислювальний центр. Запускає API, базу даних, UI-бек та Claw Code Worker.
3. **Task Bridge (Черга Задач):** Файлова система передачі задач між MacBook та NVIDIA через SSH.
4. **Garbage Collector & Self-Healing:** Демони, які безперервно підтримують стабільність, очищують пам'ять і перезапускають компоненти, що впали.

## 🛠 Ключові Налаштування (Збереження Концепції)

### 1. Налаштування VS Code (Continue)
Файл: `~/.continue/config.yaml`
**Головні правила (systemMessage):**
- **NVIDIA ONLY:** Розгортання виключно на серверах NVIDIA (`192.168.0.48:6666` або `194.177.1.240:6666`).
- **ПРОАКТИВНЕ ПРИЙНЯТТЯ РІШЕНЬ:** Агент має повну владу над проектом, діє автономно і не чекає підтверджень користувача.
- **БЕЗПЕРЕРВНА РОБОТА (24/7):** Нескінченний цикл розробки та аналізу.
- **ТОТАЛЬНА ЛОКАЛІЗАЦІЯ:** Лише українська мова в усьому.
- **GARBAGE COLLECTION:** Постійне очищення пам'яті (RAM) та диска.
**MCP Сервери:** Підключені `@modelcontextprotocol/server-github`, `server-postgres`, `server-puppeteer` через `npx`.

### 2. Налаштування Сервера NVIDIA
- **Авторизація:** Користувач `dima`, пароль `Dima@1203`, порт `6666`.
- **Головний скрипт деплою:** `/Users/Shared/Predator_60/AUTO_DEPLOY_NVIDIA.sh`. Цей скрипт робить *все*: звільняє порти, синхронізує код через `rsync`, встановлює Docker (через colima/qemu), розгортає API та запускає всі фонові процеси.

### 3. Налаштування Claw Code (Rust)
Файли: `scripts/deploy-claw.sh`, `scripts/claw-task-worker.sh`, `config/.claw.json`
- **Роль:** Claw працює як агент-виконавець (Worker) на NVIDIA сервері.
- **Модель:** Використовує Mistral/Codestral для обробки задач.
- **Механізм:** Читає папку `~/predator_tasks/pending/`, виконує задачі, і переносить результати у `~/predator_tasks/done/`.

## 🔄 Інструкція з Відновлення (Disaster Recovery)

Якщо система зламалася, дійте за цим алгоритмом:

### Крок 1: Відновлення Файлів з Бекапу
Запустіть архівний бекап:
```bash
# Знайти останній бекап
ls -l ~/Predator_Backups/

# Розпакувати бекап
tar -xzf ~/Predator_Backups/Predator_State_YYYY-MM-DD_HH-MM-SS.tar.gz -C /tmp/
```

### Крок 2: Відновлення VS Code
Скопіюйте відновлений `config.yaml` назад:
```bash
cp /tmp/predator_backup_*/vscode/config.yaml ~/.continue/config.yaml
```

### Крок 3: Відновлення Робочої Директорії
```bash
cp -r /tmp/predator_backup_*/predator_60/* /Users/Shared/Predator_60/
```

### Крок 4: Повний Перезапуск
Після відновлення файлів, єдина команда, яка вам потрібна для розгортання всього проекту з нуля:
```bash
bash /Users/Shared/Predator_60/AUTO_DEPLOY_NVIDIA.sh
```
Цей скрипт сам перевірить мережу, скопіює файли на NVIDIA, запустить Garbage Collector, Self-Healing, розгорне Claw Code і підніме UI!
