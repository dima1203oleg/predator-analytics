## 🔍 Як перевірити статус на Linux сервері

### Скрипт для моніторингу
Використовуйте скрипт `scripts/check_databases.sh` для перевірки статусу баз даних:

```bash
# Перевірка на локальній машині
./scripts/check_databases.sh

# Перевірка на віддаленому сервері
./scripts/check_databases.sh 194.177.1.240
```

### Вручну перевірка
```bash
# Підключитися до Linux сервера
ssh user@194.177.1.240

# Перевірити статус Docker контейнерів
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

# Перевірити статус баз даних
docker-compose ps
```

## 🚀 Як запустити бази даних

### На Linux сервері (NVIDIA)
```bash
# Підключитися до сервера
ssh user@194.177.1.240

# Перейти в директорію проекту
cd /path/to/Predator_60

# Запустити всі сервіси
docker-compose -f docker-compose.yml up -d

# Перевірити статус
docker-compose ps
```

### Перевірка після запуску
```bash
# Перевірити логи
docker-compose logs -f

# Перевірити статус кожної бази
docker-compose logs postgres
docker-compose logs neo4j
docker-compose logs redis
```

## 📊 System Memory Contract

Кожна база даних має свою жорстку роль:

1. **PostgreSQL (SSOT)**: "Хранитель Істини" — метадані, користувачі, фінансові реєстри
2. **ClickHouse (OLAP)**: "Аналітичний Мозок" — агрегації, історичні дані, великі масиви (100M+)
3. **OpenSearch (Search)**: "Текстова Розвідка" — keyword/hybrid пошук по документах
4. **Qdrant (Vector)**: "AI Пам'ять" — вектори для RAG та семантичного пошуку
5. **Neo4j (Graph)**: "Детектор Зв'язків" — схеми власності, фрод-ланцюжки, multi-hop аналіз
6. **Redis (Cache)**: "Швидка Пам'ять" — короткострокові дані, черги, сесії
7. **MinIO (S3)**: "Фізичне Сховище" — всі файли, скани, PDF

## ⚠️ Важливі примітки

- **MacBook** використовується тільки для розробки (IDE + Frontend)
- **Всі бази даних** запускаються на NVIDIA сервері
- **Backend API** (порт 8000) запускається на NVIDIA сервері
- **Frontend** (порт 3030) запускається на MacBook
