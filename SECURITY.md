# Безпека — Рекомендації (Predator Analytics v25.0)

Критично: цей файл містить кроки для негайних дій у разі виявлення секретів в репозиторії.

## Дія при виявленні секретів
1. **Ротація**: негайно змініть/відключіть ключі/паролі на провайдері сервісів (Groq, Gemini, GitHub тощо).
2. **Видалення з індексу та .gitignore**:

```bash
# Видалити файл з індексу
git rm --cached dynamic_keys.json
# Додати файл до .gitignore
echo "**/dynamic_keys.json" >> .gitignore
```

3. **Очищення історії**: Використовуйте `git-filter-repo` для повного видалення чутливих даних з історії коммітів.

## Рекомендації
- Використовувати **HashiCorp Vault** або хмарні сервіси управління секретами.
- Додавати **secret-scan** (gitleaks) в CI пайплайни.
- Налаштувати ліміти та біллінг-алерти для API-провайдерів.
- Використовувати змінні середовища через `.env` (ніколи не коммітити файл `.env`).

---

## Контакти та реакція на події
- У разі виявлення витоку — негайно повідомити адміністратора системи.
- Зафіксувати дату витоку, перелік скомпрометованих ключів та сервіси, що постраждали.
- Ротація ключів є обов'язковою умовою перед відновленням роботи сервісів.
