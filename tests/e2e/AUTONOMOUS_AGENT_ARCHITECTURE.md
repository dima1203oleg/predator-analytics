## 📋 Компоненти

### 1. Deployment Manager

**Файл:** `tests/e2e/deployment_manager.py`

**Функціонал:**
- SSH деплой на NVIDIA сервер (194.177.1.200)
- Перевірка статусу K8s кластера
- Автоматичний rollback при помилках
- Моніторинг ресурсів (CPU, RAM, GPU)

**Ключові методи:**
```python
class DeploymentManager:
    async def deploy_to_nvidia(self) -> bool
    async def check_cluster_status(self) -> ClusterStatus
    async def rollback_deployment(self) -> bool
    async def monitor_resources(self) -> ResourceMetrics
```

### 2. RBAC Test Suite

**Файл:** `e2e/rbac_autonomous.spec.ts`

**Функціонал:**
- Тестування 4 рівнів доступу
- Перевірка ізоляції даних
- Валідація прав доступу
- Аудит UI-гвардів

**Тести для кожного рівня:**
- **Level 1 - Admin (Tech Admin):**
  - Технічна панель
  - Керування ETL
  - Керування базами даних
  - Керування AI
  - Журнали та моніторинг
  - ЗАБОРОНА клієнтських даних

- **Level 2 - Commander (Командир):**
  - Чат
  - Голосова взаємодія
  - AI-аватар
  - Графові зв'язки
  - Аналітичні панелі
  - Історія запитів
  - Генерація звітів

- **Level 3 - Strateg (Стратег):**
  - Сценарний аналіз
  - Прогнозування
  - Побудова зв'язків
  - Семантичний пошук
  - AI-консультант

- **Level 4 - Operations Officer (Оперативний офіцер):**
  - Швидкі запити
  - Пошук
  - Картки контрагентів
  - Ризики
  - Митні декларації
  - Інтерактивні панелі

### 3. ETL Pipeline Validator

**Файл:** `e2e/etl_validator.spec.ts`

**Функціонал:**
- Excel імпорт через DOM (без API)
- Повний ETL конвеєр
- Перевірка 8 сховищ згідно з System Memory Contract
- Консистентність даних

**Перевірка сховищ:**
- **PostgreSQL (SSOT):** Метадані, користувачі, фінансові реєстри
- **ClickHouse (OLAP):** Агрегації, історичні дані, великі масиви
- **OpenSearch (Search):** Повнотекстовий пошук
- **Qdrant (Vector):** Вектори для RAG
- **Neo4j (Graph):** Схеми власності, фрод-ланцюжки
- **Redis (Cache):** Короткострокові дані, черги
- **MinIO (S3):** Файли, скани, PDF
- **Redpanda (Kafka):** Події, повідомлення

### 4. AI Workflow Tester

**Файл:** `e2e/ai_workflow.spec.ts`

**Функціонал:**
- AI-чат тести
- Перевірка походження відповідей
- Векторизація та семантичний пошук
- Графовий аналіз

**Тестові запити:**
- "Покажи декларації за березень 2024 року"
- "Які компанії імпортували найбільше товарів?"
- "Побудуй граф зв'язків контрагентів"
- "Знайди потенційні аномалії"
- "Покажи семантично схожі декларації"
- "Побудуй короткий аналітичний звіт"

**Перевірка походження відповідей:**
- Джерела даних
- Сервіси, що брали участь
- Механізм пошуку (реляційний, повнотекстовий, графовий, векторний)

### 5. Self-Healing Engine

**Файл:** `tests/e2e/self_healing_engine.py`

**Функціонал:**
- Аналіз логів
- LLM-діагностика (через локальний LLM)
- Автоматичні виправлення
- Перевірка після виправлень

**Алгоритм самовідновлення:**
1. Локалізація проблеми
2. Аналіз журналів
3. LLM-діагностика
4. Генерація патчу
5. Застосування виправлення
6. Повторне тестування
7. Збереження історії змін

**Ліміти:**
- Максимальна кількість ітерацій: 10
- Максимальний час на самовідновлення: 30 хвилин
- Зупинка при досягненні ліміту

### 6. Continuous Improvement

**Файл:** `tests/e2e/continuous_improvement.py`

**Функціонал:**
- Аналіз продуктивності
- Оптимізація SQL запитів
- Рекомендації щодо архітектури
- Автоматичний цикл регресії

**Метрики:**
- Час завантаження файлу
- Час ETL обробки
- Час валідації
- Латентність AI-відповідей
- Використання CPU, RAM, GPU

### 7. Report Generator

**Файл:** `tests/e2e/report_generator.py`

**Функціонал:**
- JSON звіти
- Markdown звіти
- HTML звіти
- Історичні дані

**Структура звіту:**
- Результати імпорту
- Результати ETL
- Перевірка кожного сховища
- Векторизація
- Графовий аналіз
- AI-відповіді
- Продуктивність
- DOM-аудит
- Інтеграційний аудит
- Список помилок
- Внесені виправлення
- Запропоновані оптимізації
- Фінальний статус PASS / FAIL
