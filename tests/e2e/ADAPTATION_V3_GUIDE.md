## 📋 Огляд адаптації

Цей документ описує адаптацію існуючого автономного агента v2.0 під нове детальне технічне завдання (ТЗ) для автономного E2E-тесту імпорту Excel через користувацький веб-інтерфейс.

## 🎯 Нові вимоги ТЗ v3.0

### 1. Симуляція користувача через Playwright
- Авторизація під користувацьким акаунтом
- Перехід до сторінки імпорту
- Завантаження Excel через DOM (input[type=file])
- Натискання кнопки «Імпорт»
- Спостереження за прогресом через UI
- Контроль WebSocket та HTTP-запитів
- Контроль помилок JavaScript
- Очікування повного завершення ETL

### 2. Перевірка ETL конвеєру
- Читання Excel
- Визначення структури
- Валідація даних
- Очищення даних
- Усунення дублікатів
- Нормалізація форматів
- Формування DTO
- Створення метаданих
- Аудит імпорту
- Підготовка текстових представлень
- Побудова індексів
- Формування ембедингів
- Створення графових зв'язків
- Завершення асинхронних задач

### 3. Перевірка 8 сховищ

**PostgreSQL:**
- Створені записи
- Збережені всі структуровані поля
- Виконані обмеження цілісності
- Кількість рядків відповідає імпорту

**ClickHouse:**
- Аналітичні таблиці
- Агреговані представлення
- Статистичні вибірки
- OLAP-структури

**Neo4j:**
- Вузли
- Ребра
- Властивості
- Графові залежності
- Зв'язки між контрагентами

**Qdrant:**
- Створення ембедингів
- Запис векторів
- Payload
- Прив'язку до первинних записів
- Працездатність similarity search

**OpenSearch:**
- Індексація
- Повнотекстовий пошук
- Агрегації
- Фільтрацію
- Пошук за окремими полями

**Redis:**
- Кешування
- TTL
- Службові ключі
- Проміжні результати
- Pub/Sub (якщо використовується)

**MinIO:**
- Наявність оригінального XLSX
- Службових артефактів
- Контрольних сум
- Метаданих

**Redpanda:**
- Створення подій
- Доставку повідомлень
- Обробку споживачами
- Відсутність втрати повідомлень

### 4. Автоматична векторизація
- Формування текстового представлення записів
- Побудова ембедингів
- Запис у Qdrant
- Зв'язка з первинними даними
- Семантичний пошук

### 5. Перевірка AI-чату
9 тестових запитів:
1. «Покажи декларації за березень 2024 року»
2. «Скільки записів було імпортовано?»
3. «Які товари зустрічаються найчастіше?»
4. «Побудуй коротке резюме імпортованих даних»
5. «Покажи найбільших імпортерів»
6. «Знайди схожі декларації»
7. «Побудуй граф зв'язків контрагентів»
8. «Знайди можливі аномалії»
9. «Покажи записи з конкретним номером декларації»

Валідація відповідей:
- Інформація відповідає фактичним даним з Excel
- Враховано нові записи після імпорту
- Відсутні застарілі або кешовані відповіді
- Результати збігаються з SQL/графовими/семантичними запитами

### 6. DOM та UX аудит
- Відображення прогресу імпорту
- Історія імпортів
- React-стан
- WebSocket-оновлення

- Коректне відображення статусів

Порівняння:

- Кількість документів у OpenSearch
- Кількість векторів у Qdrant
- Кількість вузлів та зв'язків у Neo4j
- Кількість об'єктів у MinIO
- Кількість подій у Redpanda

### 8. Самовідновлення

Тест вважається успішним лише за одночасного виконання всіх умов:

## 🔧 Зміни в існуючих компонентах

### 1. Playwright тест (autonomous-excel-import.spec.ts)
**Оновлення інтерфейсів:**
- `AIQueryResult` - додано поля для валідації відповідей
- `DOMAuditResult` - додано поля для React стану та винятків
- `consistency_check` - додано поле `differences` для розбіжностей
- `TestReport` - додано поле `ready_for_production`

### 2. Валідація 8 баз даних (validate_8_dbs.py)
**Розширення кожної функції валідації:**

**PostgreSQL:**
- Перевірка структури таблиці (nullable, default)
- Перевірка індексів з визначеннями
- Перевірка обмежень цілісності (unique, foreign keys)
- Перевірка обов'язкових полів
- Перевірка розміру таблиці
- Перевірка останнього запису
- Перевірка валідації обмежень

**Redis:**
- Перевірка TTL для ключів
- Перевірка пікової пам'яті
- Перевірка Pub/Sub каналів
- Перевірка кешованих даних
- Детальний аналіз черг та потоків

**Qdrant:**
- Перевірка загальної кількості векторів
- Перевірка індексації векторів
- Перевірка payload та його ключів
- Перевірка прив'язки до первинних записів
- Перевірка similarity search

**Neo4j:**
- Перевірка властивостей вузлів
- Перевірка графових залежностей (зв'язки між контрагентами)
- Перевірка глибини графу
- Перевірка щільності графу

**OpenSearch:**
- Перевірка мапінгу індексів
- Перевірка повнотекстового пошуку
- Перевірка агрегацій
- Перевірка загальної кількості документів

**MinIO:**
- Класифікація об'єктів (Excel vs артефакти)
- Перевірка метаданих об'єктів
- Перевірка ETag та Content-Type
- Детальна статистика по бакетах

**ClickHouse:**
- Перевірка загальної кількості таблиць
- Перевірка OLAP-структур (MergeTree)
- Перевірка статистичних вибірок
- Перевірка загальної кількості рядків

**Redpanda:**
- Тест відправки повідомлення
- Перевірка кількості топіків та партіцій
- Перевірка фактору реплікації
- Перевірка підписки на топіки

### 3. Нові компоненти

**report_generator.py:**
- Генерація звітів у форматах HTML, JSON, Markdown
- Детальна статистика та критерії успіху
- Красиве HTML оформлення з темною темою

**ai_chat_validator.py:**
- 9 тестових запитів згідно з ТЗ
- Валідація джерел відповідей
- Перевірка використання імпортованих даних
- Перевірка відповідності Excel даним
- Перевірка відсутності кешованих відповідей

**consistency_checker.py:**
- Порівняння кількості записів між сховищами
- Розрахунок розбіжностей (абсолютні та відсоткові)
- Перевірка толерансу (5%)
- Ідентифікація проблем з консистентністю
- Генерація звіту про консистентність
