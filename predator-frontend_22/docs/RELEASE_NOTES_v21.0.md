## 1. Overview

Predator Analytics v21.0 focuses on closing the **Self-Learning Loop** by introducing automated model fine-tuning (LoRA) and enhancing multi-tenant capabilities for Enterprise SaaS readiness.

---

## 2. Planned Features

### 2.1. LoRA Trainer Pipeline (Epic 1)
- **Automated Fine-Tuning Job:**
  - Ingests JSONL datasets generated by v20.0's Self-Learning Loop.
  - Trains LoRA adapters on local GPU nodes.
  - Archives adapters to persistent storage/S3.
- **Training Registry:**
  - New `lora_training_runs` table tracks every training job, its configuration, metrics (loss), and outcome.

### 2.2. SaaS Hardening (Epic 2)
- **Multi-Tenancy:**
  - Explicit `tenant_id` enforcement across all API endpoints and database records.
- **Quota Management:**
  - Database schema updates to support tiered plans (Free, Pro, Enterprise).
  - Enforced limits on API usage per tenant.

### 2.3. Admin UX 2.0 (Epic 3)
- **Enhanced Data Inspection:**
  - Detailed modal views for training samples in Admin Dashboard.
- **Manual Control:**
  - "Export Now" button to trigger dataset generation on demand.
- **Advanced Filtering:**
  - Filter usage and training data by Tenant ID and specific date ranges.
