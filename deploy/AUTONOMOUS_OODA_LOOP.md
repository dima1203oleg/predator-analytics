# Sovereign Autonomous Agent (Infinite OODA Loop)

Цей документ описує, як запустити та керувати фоновим автономним агентом (LLM Council), який відповідає за постійне вдосконалення PREDATOR Analytics без вашої прямої участі (без натискання "Accept").

## Запуск в Docker (Рекомендовано для iMac / NVIDIA Server)

Оскільки цей процес повинен працювати постійно, ми додали сервіс `autonomous-agent` до основного `docker-compose.yml`.

### 1. Налаштування `.env`
Переконайтеся, що у вашому файлі `.env` або `.env.remote` (на сервері) вказані токени:

```env
# Telegram Bot для сповіщень
TELEGRAM_BOT_TOKEN="ваш_токен_від_botfather"
TELEGRAM_CHAT_ID="ваш_chat_id"

# Налаштування LLM (LiteLLM)
OPENAI_API_KEY="..." # або інші провайдери
```

### 2. Запуск сервісу
Щоб запустити тільки агента (якщо інші сервіси вже працюють):
```bash
cd deploy/
docker-compose up -d autonomous-agent
```

Щоб переглянути логи роботи:
```bash
docker-compose logs -f autonomous-agent
```

## Як це працює?
1. **Сканування:** Агент аналізує кодову базу, логи та результати E2E тестів (Playwright).
2. **Планування:** За допомогою LangGraph формується план виправлення багів або оптимізації UI.
3. **Кодування:** Створюється код і локально зберігається.
4. **Тестування:** Запускаються тести. Якщо успішно - зміни комітяться (`git commit`).
5. **Сповіщення:** Ви отримуєте повідомлення в Telegram про те, що саме було вдосконалено.

## Зупинка
Щоб зупинити автономний цикл (наприклад, для ручного втручання):
```bash
docker-compose stop autonomous-agent
```
Або надішліть команду `/stop` вашому Telegram-боту (якщо ви додасте такий обробник).
