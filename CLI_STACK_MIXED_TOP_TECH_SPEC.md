# 🧠 ТЕХНІЧНЕ ЗАВДАННЯ: Змішаний ТОП CLI-стек для Predator Analytics

(Planning · Codegen · Review · Fallback · Offline-First)

---

## 1️⃣ МЕТА
Побудувати єдиний CLI-стек, у якому кожен інструмент виконує чітку роль, без дублювання і хаосу:
- Планування і аналіз
- Генерація коду
- Перевірка та правки
- Fallback при відмовах
- Робота без платних підписок за замовчуванням
- Підтримка LOCAL / SERVER режимів

CLI-стек = пульт керування розробкою, а не чат.

---

## 2️⃣ ПРИНЦИП АРХІТЕКТУРИ (КАНОН)
❗ **Один етап — один інструмент**
❗ **CLI не приймають рішень, а виконують ролі**
❗ **Fallback — явний, не тихий**

---

## 3️⃣ СКЛАД ЗМІШАНОГО ТОП CLI-СТЕКУ

### 🧠 Рівень 1 — Планування / Аналіз: Gemini CLI (Google)
- **Роль:** архітектор, аналітик, декомпозитор
- **Статус:** ✅ Free tier
- **Використання:** ТЗ, аналіз логів, план задач

### ✋ Рівень 2 — Генерація коду: Mistral Vibe CLI
- **Роль:** швидка генерація і правки коду
- **Статус:** ✅ Free / Open-source
- **Використання:** Python, Bash, YAML, JSON

### 🎛️ Рівень 3 — Ревʼю та правки: Aider (Open-source)
- **Роль:** code review, refactor, edge-cases
- **Статус:** ✅ 100% безплатний
- **Працює з:** Ollama / CodeLlama / DeepSeek

### 🧱 Рівень 4 — Офлайн fallback: Ollama CLI + локальні моделі
- **Роль:** офлайн-план + офлайн-код
- **Статус:** ✅ 100% free
- **Використовується при:** відсутності інтернету, лімітах API, тестах автономності

### 🎚️ Рівень 5 — Опціональний комфорт: GitHub Copilot CLI (OPTIONAL)
- **Роль:** UX-зручність
- **Статус:** ❌ Платний (не входить у обовʼязковий стек)

---

## 4️⃣ РОЗПОДІЛ РОЛЕЙ (ЧІТКО)

| Етап | CLI | Що робить | Що НЕ робить |
| :--- | :--- | :--- | :--- |
| **План** | Gemini CLI | аналіз, стратегія | не пише великі файли |
| **Код** | Vibe CLI | генерує код | не вирішує архітектуру |
| **Ревʼю** | Aider | перевіряє, править | не планує |
| **Fallback** | Ollama | автономна робота | не продакшн |
| **UX** | Copilot | підказки | не критичний |

---

## 5️⃣ ENV-AWARE ПОВЕДІНКА (ОБОВʼЯЗКОВО)
`EXECUTION_ENV=local | server`

### LOCAL
- ✅ Gemini CLI, Vibe CLI, Aider, Ollama
- ❌ GPU, Orchestrator

### SERVER
- ✅ Усе вище + Heavy tasks (через backend)
- ❌ CLI не запускає прод-операції напряму

---

## 6️⃣ СЦЕНАРІЙ ВИКОРИСТАННЯ (КАНОН)

1. **План:** `gemini "Розбий ingestion pipeline на етапи і ризики"`
2. **Генерація:** `vibe "Згенеруй Python worker для ingestion з retry"`
3. **Ревʼю:** `aider app/worker.py`
4. **Fallback:** `ollama run codellama`

---

## 7️⃣ FALLBACK-ЛОГІКА (STRICT)
Fallback активується ТІЛЬКИ якщо API недоступне, перевищені ліміти або сервіс down.
CLI зобовʼязаний вивести:
`⚠️ FALLBACK MODE: Gemini → Ollama`

---

## 8️⃣ ІНТЕГРАЦІЯ З PREDATOR CLI
Усі CLI використовуються через агентів Predator:
- `predator agents run planner --tool gemini`
- `predator agents run codegen --tool vibe`
- `predator agents run review --tool aider`

---

## 9️⃣ ЗАБОРОНИ (ОБОВʼЯЗКОВО)
❌ симуляція відповідей
❌ “розумні” відповіді без виконання
❌ silent fallback
❌ використання CLI як чат-бота
❌ обхід агентів Predator

---

## 🔟 DEFINITION OF DONE
Змішаний CLI-стек вважається готовим, якщо:
- Кожен CLI має чітку роль
- Free-first підхід працює
- Fallback перевірений
- LOCAL / SERVER не плутаються
- CLI реально допомагають розробці

---

## 11️⃣ ФІНАЛЬНИЙ ПРИНЦИП
**Не один “чарівний CLI”, а злагоджений ансамбль інструментів.**
