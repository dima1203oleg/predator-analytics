- порушувати DDD;
- порушувати Hexagonal Architecture;
- створювати циклічні залежності;
- створювати дублікати бізнес-логіки;
- порушувати API Contracts;
- порушувати Event Contracts;
- змінювати Enterprise Data Model без затвердження;
- змінювати Core Ontology без Approval Gate;
- використовувати закриті бібліотеки без погодження;
- зберігати секрети в коді;
- ігнорувати автоматичні тести;
- деплоїти неперевірений код.

#### DEFINITION OF DONE
Будь-який модуль вважається завершеним лише якщо:
* відповідає бізнес-вимогам;
* відповідає архітектурним принципам;
* проходить усі автоматичні перевірки;
* має повну документацію;
* має OpenAPI та gRPC-контракти;
* має Unit, Integration і Contract тести;
* має покриття тестами не менше 85%;
* підтримує Observability;
* не порушує AI Constitution;
* не створює технічний борг;
* забезпечує зворотну сумісність або керовану міграцію;
* готовий до безпечного розгортання в production.
