## РОЗДІЛ 1. АРХІТЕКТУРНА ПАРАДИГМА: SOVEREIGN HEADLESS (v3.0)

Система працює як абсолютно ізольований, автономний розвідувально-аналітичний комплекс, здатний до самовдосконалення через закритий цикл OODA (Observe, Orient, Decide, Act).

### 1.1. Базові Інфраструктурні Принципи
1. **Zero-Terminal**: Управління всім кластером, моніторинг інфраструктури, управління базами даних, інгестія та налаштування AI-агентів здійснюються **виключно через Web UI** (Sovereign Command Center). 
2. **Мережева Топологія (Tunnels & Nodes)**:
   - **Local IDE (Zero-Deployment)**: MacBook (`dima-mac`). Використовується виключно як термінал для написання коду (IDE) та білду UI-частини. ЗАБОРОНЕНО розгортати локально бази даних чи кластери K8s.
   - **Sentinel/Compute Node**: iMac (`192.168.0.200`). Головна точка входу, де розміщуються всі первинні БД, локальні LLM та API.
   - **Cloud Fallback**: NVIDIA Server (`194.177.1.240`). Використовується для масштабування та екстремальних обчислень.
   - **Тунелювання**: Зв'язок між вузлами та доступ до системи ззовні реалізується **виключно через захищені тунелі Zrok** (сервіс `mac_zrok_sentinel`). Будь-яке використання Ngrok або Cloudflare Tunnel СУВОРО ЗАБОРОНЕНО.

### 1.2. LLM Tri-State Routing та VRAM Guard
Маршрутизація інтелектуальних запитів відбувається у трьох станах залежно від наявної відеопам'яті (VRAM) на вузлі:
- **VRAM Guard Contract (8GB Limit)**: З 8 ГБ доступної VRAM рівно 2.5 ГБ завжди зарезервовано під WebGL/UI (GSAP/Framer), 5.5 ГБ виділяється під Local LLM Pool (Ollama). Перевищення ліміту блокується на рівні ядра.
- **SOVEREIGN Mode (Червоний)**: 100% Локальна обробка. Дані не залишають контур системи. Використовуються `Nemotron-Cascade-2` (30B MoE) для логіки та `Qwen3-Coder-Next` для генерації коду. Найвищий рівень секретності (L7).
- **HYBRID Mode (Зелений)**: Локальні моделі + швидкі хмарні API (Gemini Flash, Groq). Вмикається автоматично, якщо локальна VRAM перевищує ліміт, але запит не містить чутливих PII (Personal Identifiable Information) даних.
- **CLOUD Mode (Синій)**: Екстремальна швидкість та глибоке планування. Делегування стратегічних задач на великі хмарні моделі (Gemini 1.5 Pro, GLM-5.1, Azure Fallback).

## РОЗДІЛ 2. SYSTEM MEMORY CONTRACT (РОЗПОДІЛ ДАНИХ)

Система налічує 8 різних сховищ даних. Кожна база даних має свою жорстку роль. Перетин ролей або використання БД не за призначенням вважається технічним боргом і є блокером для прийняття PR.

1. **PostgreSQL 16 (SSOT - Single Source of Truth)**: "Хранитель Істини". Метадані системи, профілі користувачів, WORM-таблиці рішень (`decision_artifacts`, `audit_log`), фінансові реєстри та транзакції.
2. **ClickHouse**: "Аналітичний Мозок". OLAP-аналітика, важкі агрегації, сховище для мільярдних таблиць митних декларацій та логів телеметрії інфраструктури.
3. **OpenSearch 2.12**: "Текстова Розвідка". Повнотекстовий пошук, Hybrid-пошук по індексованих документах та логах чатів. ЗАБОРОНЕНО використовувати як Primary DB.
4. **Qdrant 1.8**: "AI Пам'ять". Довгострокова пам'ять агентів (Long-term Memory), векторизація ембедингів для RAG (Retrieval-Augmented Generation).
5. **Neo4j 5 (APOC + GDS)**: "Детектор Зв'язків". Графова база даних. Управління графами кінцевих бенефіціарів (UBO), виявлення ланцюжків відмивання грошей (Fraud Rings), алгоритми Shortest Path.
6. **Redis 7**: "Швидка Пам'ять". Кешування запитів (TTL 5-15 хв), управління сесіями JWT, брокер черг для легких асинхронних задач (Celery/ARQ).
7. **Kafka (Confluent 7.6)**: Шина даних (Ingestion Layer). Потокова обробка реєстрів, івент-сорсинг (Event Sourcing) для синхронізації між мікросервісами.
8. **MinIO (S3)**: "Фізичне Сховище". Зберігання PDF-документів, сканів інвойсів, згенерованих звітів, бекапів.

## РОЗДІЛ 3. СОФТВЕРНИЙ КОНТРАКТ ТА ЖОРСТКІ ПРАВИЛА (HARD RULES)

Будь-яка зміна у кодовій базі повинна проходити через цей фільтр. Порушення бодай одного правила призводить до негайного відкату змін (Rollback).

* **HR-01**: Тільки Python 3.12 для бекенду.
* **HR-02**: Типізація `Mypy strict` обов'язкова — жодних типів `Any` без явного коментаря-обґрунтування.
* **HR-03**: Коментарі, документація та назви комітів — ВИКЛЮЧНО українською мовою.
* **HR-04**: 100% UI українською. Англійська в інтерфейсі (крім брендових назв, як "Neo4j") = критична помилка.
* **HR-05**: Docker: ЗАВЖДИ multi-stage збірки, НІКОЛИ не запускати від імені root (обов'язково `USER predator`).
* **HR-06**: Секрети: НІКОЛИ не зберігати ключі (API, DB) у коді — лише як змінні середовища (env vars).
* **HR-07**: SQL: НІКОЛИ не використовувати `SELECT *` — необхідно вказувати конкретні колонки для оптимізації трафіку.
* **HR-08**: Kubernetes: Кожен Pod ЗАВЖДИ повинен мати встановлені `resource limits` (cpu + memory).
* **HR-09**: Тестування: Кожна логічна зміна повинна мати покриття тестами (Pytest для API, Vitest + RTL для DOM компонентів).
* **HR-10**: Мережа портів: Порт UI завжди **3030** | Порт Mock API завжди **9080** | Core API завжди **8000**.
* **HR-12**: Лінтер: Лише `Ruff` (`ruff.toml`), використання `flake8`/`pylint` заборонено.
* **HR-13**: Формат комітів: Тільки `feat|fix|chore|docs(scope): опис`.
* **HR-14**: Заборона використання зовнішніх залежностей/npm-пакетів, які не оновлювалися понад 1 рік.
* **HR-15**: Заборона на зовнішні SaaS для аналітики (Sentry, Google Analytics тощо). Усе має бути In-House (Loki, Grafana).
* **HR-16**: WORM-таблиці (`audit_log`, `decisions`): Операції `UPDATE` та `DELETE` на рівні БД суворо заборонені.
* **HR-17**: ClickHouse: Єдине джерело для важкої аналітики та агрегацій (>100k записів).
* **HR-18**: PostgreSQL: Лише транзакції та метадані (SSOT). Не використовувати для агрегацій > 100к рядків.
* **HR-19**: OpenSearch: Тільки повнотекстовий пошук, заборонено використовувати як Primary DB.
* **HR-20**: Qdrant: Тільки векторна пам'ять (embeddings).
* **HR-21 / HR-22**: **ZERO-LOCAL-DEPLOYMENT**. Локальний MacBook — це лише IDE. 0 запущених контейнерів баз даних локально. Весь кластер має розгортатись виключно на iMac (`192.168.0.200`) або NVIDIA Server (`194.177.1.240`).
* **HR-23**: Тунелювання ВИКЛЮЧНО через zrok (`zrok.io`). Cloudflare/Ngrok заборонені.

## РОЗДІЛ 4. UI/UX ТА ВІЗУАЛЬНА МОВА (COMMAND CENTER)

Фронтенд-модуль (`apps/predator-analytics-ui`) реалізує єдину точку входу для всього кластера — `AdminHub.tsx`. 

### 4.1. Візуальний Код (Military Cyberpunk Dark UI)
- **Фон**: Глибокий космос (`#000000`, `#050505`). Обов'язкове використання компонента `<AdvancedBackground mode="sovereign" />`.
- **Фактура (Glassmorphism)**: Напівпрозорі панелі з розмиттям (`backdrop-blur-md` до `backdrop-blur-3xl`), тонкі неонові рамки (`border-white/5`).
- **CyberGrid**: Геометрична сітка на фоні для імітації тактичного екрану (`bg-cyber-grid`).
- **Колірні Акценти (Неон)**:
  - **Cyan (`#00F5FF`) / Sky (`#0EA5E9`)**: Нормальне функціонування, нейтральна інформація, хмарні обчислення.
  - **Emerald (`#10B981`)**: Оптимальний стан, успішні операції, гібридний режим.
  - **Rose (`#F43F5E`) / Red (`#E11D48`)**: Загроза, критичні дані, Kill-Switch, локальний суверенний режим. Головний колір системи PREDATOR.
- **Типографіка**: шрифти без зарубок (sans-serif) для читабельності, `font-mono` для цифр і телеметрії. Тексти мають містити світіння (`glint-elite` клас).
- **Анімації**: Обов'язкове використання `framer-motion`. Плавні переходи (`layoutId`), пульсуючі радари, лінії серцебиття телеметрії (`HeartbeatLine`), глітч-ефекти при зміні режимів.

### 4.2. Топологія Інтерфейсу (AdminHub)
- **TopBar (Глобальний Статус Ступінь)**: Відображає логотип PREDATOR, поточний LLM-режим (Хмара/Гібрид/Локал), навантаження на CPU/VRAM, кількість активних вузлів OODA, рівень загрози L7 та системний час.
- **Category Selector (Головне Меню)**: 6 глобальних сфер застосування (SYSTEM_CORE, AI_LAB, INTEL_OSINT, BUSINESS_INTEL, PLATFORM, AUTONOMOUS_FACTORY).
- **Horizontal Tabs (Суб-модулі)**: Горизонтальна стрічка вкладок у межах обраної сфери. Плавно прокручується.
- **Workspace (Робоча Область)**: Основний простір для відображення контенту вкладки. Для специфічних модулів (напр., `AUTONOMOUS_FACTORY`) використовується додаткова вертикальна навігація зліва (`<VerticalTabNav>`).

## РОЗДІЛ 6. ПРОТОКОЛ РОЗРОБКИ ТА РОЗГОРТАННЯ (OODA 2.0 / ACP)

Незалежно від виконавця (Інженер чи Autonomous AI Agent), життєвий цикл коду відбувається за суворим конвеєром автономної роботи.

### Етап 1: Імплементація (Coding)
1. **Зрозуміти ТЗ та Топологію**: 
   - Frontend код зберігається у `apps/predator-analytics-ui/src/`.
   - Backend API — `services/core-api/`.
   - Утиліти та спільні типи — `libs/predator-common/`.
2. **Стек**: React 18, TypeScript (strict), Vite, Tailwind CSS v3, Shadcn UI.
3. **Обмеження**: Жодних зовнішніх CSS файлів (`.css`). Все стилізується через утиліти Tailwind безпосередньо у `className`.
4. **Локалізація (HR-03/HR-04)**: Усі текстові ноди, плейсхолдери, помилки пишуться українською. 

### Етап 2: Верифікація (Verification)
До відправки коду необхідно впевнитися, що він компілюється і тести проходять.
Для верифікації фронтенду (у директорії `apps/predator-analytics-ui`):
```bash
npm run typecheck    # Перевірка типів TypeScript
npm run lint         # ESLint перевірка
npm run test         # Запуск Vitest тестів (обов'язково для складних компонентів)
npm run build        # Перевірка збірки Vite
```

### Етап 3: Autonomous Commit Protocol (ACP)
Кожна атомарна зміна або завершена логічна одиниця фіксується автоматично (Для ШІ-агента активується режим `SafeToAutoRun: true`):
```bash
git add .
git commit -m "feat(module_name): детальний опис українською" --no-verify
git pull --rebase origin main
git push origin main --no-verify
```
*Використання `--no-verify` для ШІ-агента дозволяється лише для підтримки високої швидкості OODA-циклу (попередні перевірки повинні бути виконані на Етапі 2).*

### Етап 4: Розгортання (GitOps & Tunnels)
1. Компіляція та створення Docker-образів відбувається автоматично через CI (GitHub Actions) після `git push`.
2. Застосування інфраструктурних змін на Compute Node виконується виключно через скрипт:
   `bash scripts/deploy_sm.sh`
3. Усі конфігурації зберігаються у теці `deploy/helm/predator/`. ArgoCD синхронізує Helm-чарти зі станом репозиторію.
4. Доступ до розгорнутого UI забезпечується через Zrok-тунель (внутрішній порт 3030 прокидається на iMac).
