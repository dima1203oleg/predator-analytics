# Frontend UI Features (v57.0)

Цей документ описує основні фічі фронтенду PREDATOR Analytics, реалізовані в React/Vite застосунку.

## 1. Mock-Fallback Архітектура (Dual Mode)

Фронтенд спроєктовано так, щоб працювати в двох режимах, забезпечуючи безперервність роботи навіть у разі падіння основного бекенду (Zero-Local-Deployment Rule).

- **Production Mode:** Фронтенд звертається до `VITE_API_BASE_URL` (за замовчуванням `http://194.177.1.200:8000`).
- **Mock-Fallback Mode:** У разі недоступності основного API або для локальної розробки, трафік перенаправляється на `mock-api-server.mjs` (порт `9080`), який працює як локальний Express.js сервер.
- Сервер моків реалізує ідентичні ендпоінти та структуру даних, генеруючи синтетичні дані для OSINT досьє, AI відповідей та Timeline. Це гарантує безперервність роботи аналітиків (OODA loop) навіть без зв'язку з хмарною інфраструктурою.

## 2. Експорт досьє (PDF / JSON)

Платформа підтримує вивантаження сформованих розвідувальних досьє для подальшої передачі в інші правоохоронні чи аналітичні органи.

- **Експорт у JSON:** Здійснюється безпосередньо з компонента `OsintDossierPanel.tsx`. Генерується файл-зліпок поточного стейту `activeEntity`.
- **Експорт у PDF:** Реалізовано через компонент `DossierCompiler.tsx` за допомогою бібліотеки `jspdf`. Модуль захоплює структуру досьє (включаючи AI висновки, скоринг ризиків та хронологію) і генерує багатосторінковий документ формату A4.

## 3. Real-Time Timeline (Хронологія)

Вкладка **ТАЙМЛАЙН** використовує `@tanstack/react-query` для забезпечення взаємодії з бекендом у реальному часі.

- **Полінг:** Інтервал запитів на оновлення становить `10000ms` (10 секунд).
- **Динаміка:** Ендпоінт `/api/v1/osint/entity/:id/timeline` симулює надходження нових транзакцій або зв'язків з різними рівнями критичності (`LOW`, `MEDIUM`, `HIGH`, `CRITICAL`), що автоматично з'являються в UI без оновлення сторінки.

## 4. Geo-Map Інтеграція

Модуль візуалізації геолокаційних даних дозволяє відображати об'єкти на карті з використанням 3D та супутникових шарів.
- **Підтримка WebGL**: Рендеринг сотень тисяч об'єктів без втрати продуктивності завдяки Deck.gl / Mapbox.
- **Кластеризація**: Автоматичне групування об'єктів для запобігання перенавантаженню інтерфейсу.
- **Інтерактивність**: Натискання на кластер розкриває його вміст, відображаючи список об'єктів. Візуалізація теплових карт (Heatmaps) для аналізу щільності подій або фінансових транзакцій.

## 5. AI Copilot (Sovereign Headless Architecture)

Вбудований AI-асистент (Copilot) переведено з локальних Mock-заглушок на реальну взаємодію з AI Service. 

- Запити маршрутизуються через `AIService` (який може використовувати локальний Ollama або хмарний Gemini залежно від поточного статусу VRAM Guard).
- Copilot надає свої відповіді в структурованому форматі JSON, розділяючи внутрішній процес мислення (`thought_process`) та кінцеву відповідь для користувача (`answer`).
