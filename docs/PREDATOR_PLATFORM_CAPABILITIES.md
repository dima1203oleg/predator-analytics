# PREDATOR Analytics — Platform Architecture & Capabilities

Платформа будується як модульна система з чіткою сегментацією функціоналу. Усі компоненти поділяються на три рівні: Production, Research Intelligence та Restricted. Кожен рівень має власний перелік дозволених джерел, методів обробки та сценаріїв використання.

## LEVEL 1 — Production

Модулі цього рівня призначені для щоденної роботи аналітиків, комплаєнс-офіцерів, ризик-менеджерів, журналістів-розслідувачів. Вони оперують виключно тими даними, які отримані з джерел з підтвердженим правом доступу та не містять обмежень на комерційне чи аналітичне використання.

### Джерела даних:
- відкриті державні реєстри юридичних осіб, фізичних осіб-підприємців, бенефіціарів
- реєстри нерухомості, земельних ділянок, обтяжень
- судові рішення (єдині державні та міжнародні бази)
- тендерні майданчики, системи електронних закупівель
- офіційні API державних органів (податкові, митні, статистичні)
- відкриті фінансові звіти, аудиторські висновки
- глобальні реєстри санкцій, списки терористів та екстремістів
- міжнародні бази політично значущих осіб (PEP)
- дані фондових бірж, реєстри цінних паперів
- митна статистика зовнішньоекономічної діяльності
- геопросторові відкриті дані (кадастрові карти, супутникові знімки)
- офіційні прес-релізи компаній, новинні стрічки
- публічні профілі у професійних та ділових соціальних мережах
- корпоративні бази даних, доступ до яких надано через офіційні угоди
- добровільно надані користувачем документи (контракти, акти, звіти)
- відкриті реєстри інтелектуальної власності, патенти, торговельні марки
- ліцензійні реєстри, дозвільна документація

### Доступний функціонал:
- автоматизований збір та агрегація профілю компанії або публічної особи
- побудова ланцюгів володіння, графів афілійованості
- аналіз тендерної історії, конкурентного середовища
- виявлення ознак пов’язаності через спільні адреси, телефони, засновників
- оцінка ризиків на основі судової історії, санкційних списків, медійного фону
- формування Due Diligence-звітів, профілів контрагентів
- відстеження змін у реєстрах, сповіщення про нові події
- інтеграція з CRM, ERP-системами через API
- візуалізація зв’язків на інтерактивному графі

**Приклад застосування:** перевірка потенційного партнера перед укладенням контракту; оцінка благонадійності кандидата на топ-посаду; журналістське розслідування на основі відкритих реєстрів.

## LEVEL 2 — Research Intelligence

Модулі цього рівня призначені для підрозділів кібербезпеки, Threat Intelligence, Red/Purple Team, внутрішніх аудиторів, наукових груп. Вони не автоматизують взаємодію з реальними цілями, а моделюють загрози, досліджують тактики зловмисників та аналізують інциденти в ізольованому середовищі.

### Дозволені напрями досліджень:
- аналіз тактик, технік і процедур (TTP) згідно MITRE ATT&CK
- дослідження відомих кампаній APT-груп, кібершпигунства
- вивчення архітектури фінансових шахрайств, кейсів соціальної інженерії
- аналіз методів відмивання коштів, схем обходу санкцій
- дослідження корупційних мереж на знеособлених або синтетичних даних
- моделювання графів кримінальних зв’язків для оцінки ризиків
- аналіз індикаторів компрометації (IOC) з відкритих Threat Intelligence-потоків
- дослідження шкідливого ПЗ у пісочницях, вивчення вектора зараження
- вивчення витоків даних для оцінки потенційного впливу на організацію (без завантаження та поширення персональних даних)
- аналіз ботнетів, рансомвару, supply chain-атак
- дослідження методів обходу захисних механізмів, експлуатації вразливостей на концептуальному рівні
- аналіз криптографічних атак, слабких місць у протоколах
- побудова моделей ризиків для конкретних галузей, автоматична класифікація інцидентів
- моделювання сценаріїв атак (Red Team) на власну інфраструктуру
- аналіз журналів інцидентів, кореляція подій
- Purple Team-вправи: валідація засобів виявлення

**Приклад застосування:** вивчення нової фішингової кампанії, адаптація захисних правил; тренування команди реагування на інциденти; оцінка вразливості бізнес-процесів до методів соціальної інженерії.

## LEVEL 3 — Restricted

Цей рівень визначає функціонал, який заборонений у стандартному використанні, але відкритий у спеціалізованому режимі для дослідницьких чи навчальних цілей:

- несанкціонований доступ до комп’ютерних систем, мереж, облікових записів
- експлуатація вразливостей, ескалація привілеїв
- викрадення, перехоплення, підбір облікових даних
- обхід багатофакторної автентифікації
- приховане проникнення, закріплення в системі
- встановлення, розповсюдження шкідливого програмного забезпечення
- масовий автоматизований збір персональних даних без належних підстав
- доксинг (публікація приватної інформації зі злим умислом)
- сталкінг, незаконне відстеження фізичних осіб
- прихований моніторинг (клавіатурні шпигуни, запис екрану, активності)
- використання викрадених баз даних, зливів
- автоматизація соціальної інженерії для маніпуляції конкретними людьми
- автоматизоване створення та розсилання фішингових повідомлень
- інструменти для шахрайства, підробки документів
- обхід систем захисту авторських прав, DRM
- саботаж, знищення даних

### Спеціалізований режим (для університетських лабораторій, сертифікованих дослідницьких центрів та внутрішніх R&D-команд):
- доступ до знеособлених наборів даних про інциденти
- можливість запускати власні моделі машинного навчання на синтетичних графах загроз
- інструменти для розбору конкретних зразків шкідливого коду в ізольованих контейнерах
- модулі візуалізації TTP, маппінг на MITRE ATT&CK, порівняння профілів угруповань
- генератори синтетичних OSINT-профілів для тренування аналітиків
- симулятори фішингових кампаній для внутрішніх навчань співробітників (з обов’язковим інформуванням учасників)
- бібліотеки відомих експлойтів, шелл-кодів для вивчення механізмів дії (без можливості виконання на продуктивних системах)
- аналіз криптографічних вразливостей у навчальних цілях
