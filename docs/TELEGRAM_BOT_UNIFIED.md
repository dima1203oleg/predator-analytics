## 🤖 Nexus Telegram Bot - Інтелектуальний пульт управління

### Архітектура
```
Користувач (природна мова)
    ↓
NexusTelegramBot
    ↓
LLM (розуміння наміру)
    ↓
Orchestrator (розподіл задач)
    ↓
Agents (виконання)
    ↓
Arbiter (вибір кращої відповіді)
    ↓
Відповідь користувачу
```

### Можливості

#### 1. Природномовні команди
```
Користувач: "проаналізуй аномалії в імпорті електроніки за 2024"
Бот: 
  1. Розуміє намір через LLM
  2. Визначає потрібні агенти (RetrieverAgent, AnomalyAgent)
  3. Оркеструє виконання
  4. Повертає результат
```

#### 2. Автономне прийняття рішень
- Бот може самостійно приймати рішення на основі політик
- Для критичних операцій запитує підтвердження
- Використовує арбітраж між моделями для вибору кращого рішення

#### 3. Інтеграція з Nexus Core
- Доступ до всіх 16 агентів системи
- Використання 58 моделей через ModelRouter
- Моніторинг стану системи
- Запуск самовдосконалення

### Команди

#### Базові
- `/start` - Початок роботи
- `/help` - Довідка
- `/status` - Статус системи

#### Агенти та оркестрація
- `/agents` - Статус всіх агентів
- `/orchestrate <задача>` - Запуск оркестрації
- `/arbiter` - Статус арбітра моделей

#### Датасети та аналітика
- `/datasets` - Список датасетів
- `/analyze <dataset>` - Аналіз датасету
- `/anomalies` - Пошук аномалій
- `/radar <entity>` - Швидкий звіт по конкуренту (v55.2)
- `/simulate <params>` - Запуск ринкової симуляції (v55.2)
- `/forecast` - Прогнозування

#### ML та моделі
- `/models` - Статус моделей
- `/train` - Тренування моделі
- `/inference` - Запуск інференсу

#### Система
- `/health` - Стан системи
- `/metrics` - Метрики
- `/self_improve` - Самовдосконалення

### Приклади використання

#### Аналіз даних
```
/orchestrate знайди всі підозрілі імпорти з Китаю за останній місяць

Бот:
🎭 Результат оркестрації

✅ Задача виконана успішно

Використані агенти:
  • RetrieverAgent
  • AnomalyAgent
  • GraphIntelligenceAgent

Результат:
Знайдено 23 підозрілі декларації:
- 15 з аномальними цінами
- 8 з підозрілими маршрутами
...

⚖️ Оцінка арбітра: 0.94
```

#### Прогнозування
```
/forecast попит на електроніку на наступний квартал

Бот використає:
1. ForecastAgent для прогнозу
2. Кілька моделей (Prophet, LightGBM, XGBoost)
3. Arbiter для вибору кращого прогнозу
```

#### Самовдосконалення
```
/self_improve

🔄 Самовдосконалення запущено

Режим: auto
Етапи:
  1. ✅ Діагностика
  2. 🔄 Аналіз дрейфу
  3. 🔄 Генерація датасетів
  4. 🔄 Тренування моделей
  5. ⏳ Валідація
```

## 🔐 Безпека

### Авторизація
- Тільки користувачі з `TELEGRAM_ADMIN_ID` мають повний доступ
- Критичні операції вимагають підтвердження
- Всі дії логуються в аудит

### PII Protection
- Чутливі дані маскуються за замовчуванням
- Розкриття вимагає спеціальних прав
- Всі розкриття записуються в `audit_pii_access`
