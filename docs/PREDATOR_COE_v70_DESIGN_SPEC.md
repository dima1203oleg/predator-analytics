# PREDATOR Analytics COE v7.0 (Cognitive Operating Environment) — Дизайн-специфікація

Ти — UI/UX Designer, Visual Design Lead, Motion Designer, 3D Graphics Designer, Information Architect та Brand Strategist.
Твоя задача — створити цілісний дизайн веб-інтерфейсу PREDATOR Analytics COE v7.0 (Cognitive Operating Environment) — простору спостереження за процесом міркування штучного інтелекту.

Це не звичайний сайт. Це середовище, де аналітичні процеси, побудова зв’язків, перевірка гіпотез і формування висновків AI візуалізовані як фізичне явище. Тут немає антропоморфних аватарів. Інтелект системи виражено через геометрію, фізику, світло, енергію, пам’ять та увагу. Інтерфейс не показує «свідомість» — він робить видимим процес обробки даних, логічних переходів і аналітичних рішень.

Дизайн повинен бути повністю готовим до втілення — з системою кольорів, типографікою, просторовою композицією, правилами руху, бібліотекою семантичних об’єктів, матеріалами, освітленням, станами UI, сценаріями взаємодії та вимогами доступності.

Цей документ є дизайн-специфікацією. Конкретні технічні обмеження (FPS, кількість шейдерів тощо) описані в окремій Engineering Specification. Дизайн-специфікація описує лише візуальні та UX-правила, які повинні масштабуватися відповідно до доступної продуктивності без втрати семантики.

## 1. СУТНІСТЬ ПРОДУКТУ

PREDATOR COE — простір спостереження за процесом міркування AI.
Він не розважає. Він викликає:
- відчуття повного контролю над складними даними;
- довіру до аналітичних процесів;
- повагу до системи;
- впевненість, що тут приймають обґрунтовані рішення.

Жодної театральності, «хайтеку заради хайтеку», ігрової естетики чи дешевого Sci‑Fi.
Усе, що рухається, має причину. Усе, що світиться, має значення.

## 2. UNIVERSAL COGNITIVE PHYSICS (Закони всесвіту PREDATOR)

1. Будь-який рух повинен мати причину. Ніщо не рухається без події.
2. Будь-яка анімація повинна бути наслідком події (запит, відповідь, знахідка, помилка).
3. Будь-яке світіння повинно означати зміну знань (нове, підтверджене, сумнівне, небезпечне).
4. Будь-яке зникнення повинно означати втрату інформації (застарівання, спростування, архівація).
5. Будь-яка деформація означає процес мислення.
6. У системі не існує декоративної анімації. Кожен рух семантичний.
7. Інформація завжди важливіша за красу. Жоден ефект не заважає читанню даних.
8. Кожне повідомлення повинно мати контекст.
9. Користувач ніколи не втрачає орієнтацію.
10. Немає різких рухів. Весь рух прогнозований.

## 3. ВІЗУАЛЬНА МОВА (Visual Language Bible)

### 3.1 Семантична система кольорів
Кожен колір має єдине значення на всіх рівнях системи:

| Колір | Значення |
|-------|----------|
| Cyan (#4A90D9) | Пошук, дослідження, активна навігація |
| Blue (#2D5F8A) | Стабільні знання, підтверджена інформація |
| Green (#3A7D44) | Підтверджені факти, верифіковані дані |
| Amber (#D69E2E) | Логічний процес, роздуми, незавершений аналіз |
| Purple (#7B4FBF) | Конфлікт даних, суперечність, альтернативна гіпотеза |
| Red (#9B2C2C) | Ризик, загроза, критичне сповіщення |
| White (#E2E8F0) | Нове відкриття, інсайт, прорив |
| Black / Deep Navy (#0A1128) | Відсутність інформації, фон, глибина |

### 3.2 Заборонено
- Неонові кольори, перенасичені відтінки.
- Дзеркальні поверхні.
- Постійний bloom, glitch без причини.
- Градієнти «веселка».
- Декоративні шрифти.

### 3.3 Типографіка
- **Основний шрифт:** Inter (sans-serif) — інтерфейс, підписи, кнопки.
- **Моноширинний:** JetBrains Mono — числові дані, таблиці.
- Розміри (rem): Заголовки 1.75/1.5/1.25; текст 1; дрібний 0.875; мікропідписи 0.75.
- Вага: Regular, Medium, SemiBold. Bold — лише для критичних сповіщень.

### 3.4 Простір та сітка
- Базовий юніт: 8px.
- Відступи: 16–24px.
- Радіуси: 4px (картки), 8px (модальні), 2px (кнопки).
- Тіні: мінімальні, лише для DOM-шарів (HUD).

### 3.5 Іконографія
- Лінійні іконки (phosphor-style), 1.5px обведення.
- Колір: `text-secondary`, при наведенні — `accent-info`.

## 7. MOTION LANGUAGE (Граматика руху)

Кожен тип анімації має чітке семантичне значення:

| Анімація | Значення |
|----------|----------|
| Пульсація (scale 1.0 → 1.05 → 1.0) | Мислення, обробка |
| Обертання (навколо осі) | Аналіз, сканування |
| Спіраль (рух по спіралі) | Пошук, дослідження |
| Вибух (швидке розширення) | Інсайт, знахідка |
| Колапс (стискання до точки) | Відхилена гіпотеза |
| Хвиля (поширення від центру) | Поширення знань, нові зв’язки |
| Мерехтіння (opacity 0.5–1.0) | Невизначеність, сумнів |
| Дрейф (повільне зміщення) | Застарівання, архівація |

Правила:
- Одночасно активно не більше 3 типів анімацій.
- Тривалість переходів: 0.4–1.2 с, easeInOutCubic.
- Пульсація — не частіше ніж 1 раз на 2 секунди.

Кожен стан системи супроводжується комплексними змінами у фізиці, освітленні та звуці.

| Стан | Візуальна поведінка сцени | Освітлення та Погода |
|------|--------------------------|----------------------|
| Idle (Очікування) | Граф повільно дрейфує. Ядро ледь дихає. | Calm (Синій, м'який) |
| Receiving (Збір) | Хвилі проходять по графу, з'являються нові вузли. | Exploration (Золотавий) |
| Reasoning (Аналіз) | Швидке обертання Ядра, утворення кластерів. | Thinking (Теплий жовтий) |

## 11. INSIGHT EXPLOSION (Кульмінація середовища)

Коли AI знаходить ключове рішення, запускається послідовність:
1. Ядро стискається (0.5 с).
2. Енергія накопичується — світіння посилюється.
3. Усі зв’язки завмирають — тиша.
4. Удар — вибух білого світла, розширення Ядра.
5. Нові вузли та зв’язки розлітаються від центру.
6. Погода змінюється → Stabilization.
7. Пам’ять оновлюється.
8. HUD показує підсумок.

Це ключовий «wow-момент», але стриманий і професійний.

## 12. ПРОСТОРОВА КОМПОЗИЦІЯ

Сцена поділена на ключові зони:
1. **Обчислювальне Ядро (Core)** — центр (0,0,0). Ікосаедр високої деталізації, що деформується залежно від навантаження.
2. **Кузня Порожнечі (Void Forge)** — навколо Ядра (радіус 2–3). Фізична зона, де кінематичний молот генерує іскри при аналізі.
3. **Синаптична Сітка (Synaptic Mesh)** — зовнішня сфера (радіус 5–15). Тривимірний граф знань, що динамічно розширюється.
4. **Часовий шар (Temporal Layer)** — візуалізація історії та прогнозу.
5. **Шари пам'яті (Memory Layers)** — Short-Term / Working / Long-Term / Forgetting, розташовані по осі Z.
6. **Поле уваги (Attention Field)** — світловий індикатор фокусу AI.
7. **Агентні поля (Agent Fields)** — хмари частинок для кожного AI-агента.
8. **Когнітивна погода (Cognitive Weather)** — глобальний стан середовища, що змінює освітлення, колір аури, швидкість частинок.
9. **HUD** — окремий DOM-шар поверх Canvas із телеметрією, сповіщеннями та Explainability-панеллю.

## 13. КОГНІТИВНА ПОГОДА (Cognitive Weather)

Глобальні стани, що змінюють атмосферу всієї сцени:

| Стан | Колір аури | Ефекти |
|------|-----------|--------|
| Calm | Синій, спокійний | Мінімальна активність, легке дихання Ядра |
| Exploration | Золотавий | Хвилі, прискорення частинок |
| Storm | Фіолетовий | Турбулентність, швидкі обертання |

| Insight | Білий спалах | Кульмінація |
| Stabilization | Повернення до синього | Плавне затухання ефектів |

## 14. СЕМАНТИЧНА ФІЗИКА (Semantic Physics)

Правила поведінки вузлів у графі, що випливають із їх семантики:

| Зв’язок | Поведінка |
|---------|-----------|
| Подібні документи | Притягуються (сила тяжіння більша за замовчувану) |
| Конфліктні дані | Відштовхуються (сила відштовхування більша) |
| Невизначені | Хаотично рухаються (додатковий шум до позиції) |
| Архівні | Повільно дрейфують (демпфування швидкості) |
| Підтверджені | Стають стабільними (фіксуються в просторі) |
| Ризикові | Пульсують і світяться червоним |

Ця поведінка реалізується через модифікатори сили в d3-force-3d, які конфігуруються на основі типу вузла та ребра.

## 15. СИСТЕМА УВАГИ AI (Attention System)

- **Attention Field** — напівпрозора сфера або конус світла навколо вузла, на якому зосереджений AI в даний момент.
- Камера плавно зміщується, тримаючи поле в кадрі (режим Focus Mode).
- HUD показує контекстну підказку: «AI аналізує зв’язок між X та Y».
- Частинки в зоні уваги прискорюються, їх щільність збільшується.

## 16. AI PERSONALITY (Характер системи)

Система має налаштовуваний темперамент, що впливає на візуальну поведінку:

| Тип | Швидкість анімацій | Кольорова гама | Погода | Звуковий супровід |
|-----|-------------------|---------------|--------|-------------------|
| Спокійний | Повільна | Синій | Calm | Тихий гул |
| Обережний | Середня | Зелений | Exploration | Часті перевірки |
| Агресивний | Висока | Червоний | Storm | Різкі сигнали |
| Експериментальний | Швидка | Фіолетовий | Conflict | Хаотичні тони |
| Консервативний | Повільна | Синьо-сірий | Calm | Мінімум звуків |

Перемикання темпераменту відбувається в налаштуваннях HUD.

## 17. CAMERA DIRECTOR

| Режим | Поведінка |

## 18. КОРИСТУВАЦЬКА ПОДОРОЖ (User Journey)

Типова сесія в PREDATOR COE виглядає як кінематографічний процес розкриття істини:

1. **Порожнеча:** Користувач відкриває систему. На екрані — глибокий чорний простір, повільно обертається спляче Ядро.
2. **Пробудження:** Введення запиту. Ядро спалахує ціановим світлом (State: Receiving). Починається рух.
3. **Експансія:** Дані надходять з API. Навколо Ядра починають рости гілки Синаптичної Сітки.
4. **Фокус AI (Attention System):** Камера автоматично масштабується на конкретному кластері. HUD повідомляє: «AI аналізує зв'язок між Компанією А та Особою Б».
5. **Конфлікт:** Система знаходить розбіжність у даних. Світло змінюється на фіолетове. Вузли гіпотез починають відштовхуватися.
6. **Insight Explosion (Кульмінація):** Система знаходить прихований ризик. Секундна тиша, стискання Ядра, і раптовий спалах світла. Новий червоний кристал (Ризик) розриває попередні зв'язки.
7. **Пояснення:** Камера фокусується на інсайті. HUD виводить текстове логічне пояснення кроків AI.
8. **Рішення:** Середовище стабілізується (State: Calm), фіксуючи нову, підтверджену топологію знань у довгострокову пам'ять.

## 19. АДАПТИВНІСТЬ ТА ДОСТУПНІСТЬ

### 19.1 Адаптивні макети
- **Desktop (1440p+):** повна 3D-сцена, всі зони видимі, максимальна деталізація.
- **Laptop (1080p–1440p):** незначне зменшення кількості частинок, менший радіус графу.
- **Tablet:** дві колонки, сенсорні жести (pinch-zoom, pan), панелі згортаються.
- **Smartphone:** автоматичне перемикання в Mobile Command Mode — 3D-сцена замінена на спрощений 2D/2.5D інтерфейс. Ядро — статичне або анімоване зображення вгорі, решта — повноекранні панелі.

### 19.2 Доступність (Accessibility)
- **Color Blind Mode:** заміна семантичних кольорів на текстурні патерни (штрихування, крапки) при збереженні значень.
- **High Contrast:** підвищена контрастність між фоном та об’єктами, вимкнення Bloom, 100% непрозорість HUD.
- **Reduced Motion:** усі анімації сповільнюються або замінюються на миттєві переходи, вимикається дрейф частинок.
- **Silent Mode:** вимкнення всіх звукових ефектів, окрім критичних сповіщень.
- **Minimal Visualization:** спрощена геометрія (2D-топологія), вимкнення 3D-матеріалів та постобробки.
- **Keyboard Navigation:** можливість переміщуватись між елементами HUD та викликати основні дії без миші.

## 20. UX PRINCIPLES

- Одна дія → одна головна реакція.
- Не більше двох нових ефектів одночасно.
- Інформація завжди важливіша за красу.
- Кожне повідомлення має контекст.
- Користувач ніколи не втрачає орієнтацію.
- Немає різких рухів камери.
- Весь рух прогнозований.

## 21. РЕЗУЛЬТАТ РОБОТИ ДИЗАЙНЕРА

На виході мають бути:

1. **UI Kit (Figma)** з усіма компонентами у трьох темах (Professional Matte, Stealth, Briefing).
2. **Семантична Бібліотека Об’єктів** — візуальні представлення всіх типів даних (документи, організації, особи, ризики, події, гіпотези) у різних станах (Idle, Selected, Conflict, Insight).
3. **Адаптивні макети** ключових екранів:
   - Desktop: повна сцена з HUD.
   - Laptop: дещо спрощена сцена.
   - Tablet: двоколонковий режим із сенсорними жестами.
   - Mobile: Mobile Command Mode із повноекранними панелями.
4. **Інтерактивний прототип** основних сценаріїв:
   - Початок сесії (Порожнеча → Пробудження).
   - Збір даних та експансія графу.
   - Конфлікт даних (фіолетове світло, відштовхування вузлів).
   - Insight Explosion (спалах, поява нових зв'язків).
   - Перемикання між станами UI (Idle, Reasoning, Alert, Completed).
   - Робота з Explainability Engine (відкриття панелі інспектора).
5. **Motion Design Specification** — тривалості, easing curves, хореографія появи елементів, правила пульсації, обертання, спіралей, вибухів.
6. **Visual Language Bible** — повний опис системи кольорів, типографіки, іконографії, матеріалів, освітлення.
7. **Interaction Language** — стандартизовані жести та візуальні відгуки на них.
