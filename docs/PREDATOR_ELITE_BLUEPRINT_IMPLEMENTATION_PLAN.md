## Фаза 1: Оновлення навігації та статусів (Пріоритет: ВИСОКИЙ)

### 1.1 Додати позначки статусів до навігації
**Файл**: `apps/predator-analytics-ui/src/config/navigation.ts`

Додати поле `accessLevel` до кожного NavItem:
- `terminal` (🟢) - відкрито в Terminal та вище
- `pro` (🟡) - відкрито в Pro та вище; для Terminal - пейвол
- `sovereign` (🔴) - відкрито тільки в Sovereign

Приклад:
```typescript
{
  id: 'dashboard',
  label: 'Глобальна Панорама',
  path: '/command?tab=board',
  icon: LayoutDashboard,
  description: 'Командний дешборд макроекономічних потоків та загроз.',
  accessLevel: 'terminal', // 🟢
  audiences: ['terminal', 'pro', 'sovereign'],
}
```

## Фаза 2: ABAC та Dynamic Redaction (Пріоритет: ВИСОКИЙ)

### 2.1 Створити ABAC контекст
**Файл**: `apps/predator-analytics-ui/src/context/AccessContext.tsx`

Додати атрибути:
- Геолокація користувача
- Тип контракту
- Поточний санкційний режим
- Рівень загрози
- Час доступу

### 2.2 Реалізувати Dynamic Redaction на UI рівні
**Файл**: `apps/predator-analytics-ui/src/components/common/DataMasker.tsx`

Компонент для маскування даних:
- Фінансові дані: діапазони для Terminal
- Імена бенефіціарів: часткове маскування
- Телефони/Email: токенізація
- Графи: обрізання до N рівнів

### 2.3 Інтегрувати Context-Aware Access
Додати логіку перевірки контексту перед доступом до модулів.

## Фаза 4: Тестування та валідація (Пріоритет: ВИСОКИЙ)

### 4.1 E2E тести для RBAC
**Файл**: `apps/predator-analytics-ui/e2e/rbac-scenarios.spec.ts`

Сценарії:
- Terminal користувач бачить усі модулі, але 🔴 заблоковані
- Pro користувач має доступ до 🟡 модулів
- Sovereign користувач має доступ до 🔴 модулів
- Core користувач бачить тільки технічні секції

### 4.2 Тести Dynamic Redaction
Перевірка маскування даних для різних рівнів.
