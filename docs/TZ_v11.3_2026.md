### 1. Вступна частина

#### 1.1 Передумови

Попередні версії ТЗ сформували сучасну Enterprise-архітектуру з 6 кореневими бізнес-блоками, глобальним шаром, контекстною панеллю, ROI-віджетами та центром виконання.

Аналіз показав необхідність посилити фундаментальні блоки, щоб забезпечити **максимальну довіру клієнтів**, прозорість розрахунків, надійність монетизації та масштабованість продукту.

Додаткові критичні елементи:

- Стратегія даних і якість
- Валідація економії та рівень довіри
- Гібридна модель монетизації (% від економії + підписка)
- Пояснюваність AI
- Outcome-First UX та Value Screen
- Життєвий цикл клієнта від демо до масштабування
- Go-to-Market стратегія

#### 1.2 Мета документа

Створити остаточне, детальне, готове до реалізації Технічне Завдання, яке стане основним керівним документом для команди розробки, продукту, продажів та маркетингу.

### 2. Цілі та завдання

#### 2.1 Бізнес-цілі

- Досягнення «aha moment» протягом **≤ 3 хвилин** для ключових ролей (business та supply_chain)
- Підвищення конверсії в платних користувачів на 30–40%
- Зниження churn протягом перших 30 днів на 20%
- Створення продукту, який продає себе через швидку та прозору демонстрацію реальної грошової цінності
- Позиціонування Predator Analytics як провідної операційної системи прийняття рішень в Україні та Східній Європі

#### 2.2 Технічні цілі

- Реалізувати чисту, інтуїтивну навігацію з чіткими 6 бізнес-блоками
- Забезпечити повну рольову адаптацію (RBAC) та тарифну фільтрацію
- Створити прозору систему валідації економії з рівнем довіри
- Закласти масштабовану архітектуру монетизації
- Передбачити збір даних для постійної оптимізації продукту

### 4. Ключовий бізнес-сценарій (Killer Use-Case)

**Назва:** Оптимізація закупівель імпортера  
**Цільова аудиторія:** Керівники та менеджери закупівель українських компаній-імпортерів  
**Очікувана цінність:** Показати конкретну суму економії в гривнях за один сценарій

**Основний потік у MVP:**

1. Користувач вводить назву товару / код УКТЗЕД або запускає Demo Mode
2. Система аналізує дані та формує рекомендацію
3. Відображається **Value Screen** з сумою економії, рівнем довіри та припущеннями
4. Пропонується зберегти сценарій та запустити його регулярно

-----

### 5. Outcome-First UX та Value Screen

Value Screen — це ключовий елемент продукту. Після виконання сценарію користувач бачить:

- Конкретну суму економії (грн / USD)
- Рівень довіри (confidence score 0–100%)
- Ключові припущення розрахунку
- Детальний звіт і порівняння
- Наступні рекомендовані дії
- Кнопки «Зберегти сценарій», «Запустити автоматично», «Поділитися»
- Юридичний disclaimer

-----

### 6. Модель монетизації

- **Підписки:** Basic / Pro / Enterprise
- **% від економії:** 5–8% від підтвердженої економії (основна модель для Pro та Enterprise)
- **Pay-per-Action:** оплата за складні аудити та запуск агентів
