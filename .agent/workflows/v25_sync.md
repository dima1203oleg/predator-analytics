---
description: Автоматична синхронізація V25 Analytics UI між локальною машиною та сервером
---

Цей воркфлоу забезпечує єдине джерело правди та автоматичне розгортання при відновленні зв'язку з сервером.

### Крок 1: Перевірка статусу сервера
Завжди перевіряйте, чи доступний сервер перед початком роботи.
// turbo
```bash
ssh predator-server "echo 'Server is UP'" || echo "Server is DOWN"
```

### Крок 2: Синхронізація (Deploy)
Якщо сервер онлайн, запустіть скрипт деплою. Він збере проект локально та завантажить на сервер, перезапустивши контейнер.
// turbo
```bash
/Users/dima-mac/Documents/Predator_21/scripts/deploy_v25.sh
```

### Крок 3: Верифікація версії
Після кожного деплою перевіряйте заголовок сторінки, щоб переконатися, що v25 активна.
// turbo
```bash
ssh predator-server "curl -s http://localhost:8080 | grep '<title>'"
```

### Правила захисту від застарілих версій:
1. **Ніколи** не використовуйте `docker-compose up` для фронтенду без перевірки монтування папки `dist`.
2. Всі зміни в інтерфейсі повинні проходити через `deploy_v25.sh`.
3. Якщо ви бачите "v29" або "SOM Control" — це кеш браузера або Service Worker. Використовуйте режим Incognito або Unregister Service Worker.
