# Автоматичний імпорт та синхронізація митних декларацій (Customs ETL)

// turbo-all

## Алгоритм виконання

1. **Перевірка наявності нових даних**:
   - Перевірити директорію `./data_staging/` на наявність .xlsx файлів.
   - Якщо файлів немає, перевірити основну директорію `./data/`.

2. **Підготовка середовища**:
   - Переконатися, що контейнери `predator_backend` та `predator_postgres` запущені.
   - Синхронізувати локальні зміни парсера з контейнером:
     `docker cp ./scripts/customs_parser.py predator_backend:/app/scripts/`

3. **Запуск обробки**:
   - Виконати скрипт імпорту:
     `./scripts/execute_customs_import.sh`

4. **Валідація та Синхронізація**:
   - Перевірити статус останнього завдання через API:
     `curl -X GET http://localhost:8090/api/v1/ingest/status/last`
   - Запустити примусову синхронізацію графа для нових записів:
     `python3 scripts/run_production_etl.py --sync-only`

5. **Звітність**:
   - Згенерувати звіт про аномалії за останній період.
   - Оновити статистику у Pipeline Monitor.
