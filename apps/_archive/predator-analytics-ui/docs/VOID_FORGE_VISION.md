# PROJECT: VOID FORGE - Master Vision

This document serves as the foundational design and architectural philosophy for the PREDATOR ANALYTICS interface (Project Void Forge).

## PROJECT VISION
Create an extraordinary browser-based AI interface that visualizes the internal cognitive process of an artificial intelligence rather than displaying traditional dashboards. The user should feel as if they have entered the living consciousness of a superintelligent analytical system. Every visible object must appear alive. Nothing remains static.

## CORE PHILOSOPHY
The interface represents the internal mind of the AI. It visualizes reasoning, memory, hypotheses, correlations, predictions, uncertainty, confidence, anomalies, and learning. The user is observing the thinking process itself.

## DESIGN LANGUAGE
- Visual Style: Living Intelligence, Quantum Architecture, Sci-Fi Brutalism, Minimal Futurism
- Elements: Cinematic UI, Volumetric Lighting, Dark Matter, Liquid Geometry, Dynamic Energy, Organic Computational Structures

## COLOR PALETTE
- Primary: Ultra Black, Dark Graphite, Obsidian
- Secondary: Aether Cyan, Quantum Blue, Electric Violet, AI White
- Indicators: Forge Amber (Warning), Crimson Red (Critical), Emerald Green (Success)
- Rule: All colors must emit physically-based bloom.

## USER EXPERIENCE & STARTUP
The application starts inside absolute darkness. Only a faint breathing sound is audible. A small energy pulse appears and expands into thousands of particles, forming connections. The AI consciousness is born before the user’s eyes.

## MAIN ENVIRONMENT & CENTRAL OBJECT
Infinite darkness filled with volumetric fog, floating particles, and spatial light. 
**The Quantum Cognitive Core:** A constantly morphing central object driven procedurally using GPU noise fields and custom shaders. It expands, splits, and reconnects, resembling neural networks, crystals, galaxies, or liquid-metal intelligence.

## KNOWLEDGE UNIVERSE & REASONING
Nodes (companies, shipments, documents, embeddings) float and reorganize via force-based simulation based on semantic similarity.
When the user asks a question, the reasoning sequence triggers:
1. Global silence
2. Memory activation (connections illuminate)
3. Hypothesis generation (reasoning clusters form)
4. Competition (clusters exchange info)
5. Inference (one dominant structure emerges)
6. Final synthesis (crystallization and answer display)

## CAMERA, POST-PROCESSING & HUD
Cinematic camera with inertia, micro-breathing, and soft acceleration.
Post-processing includes Bloom, SSAO, DoF, Volumetric Fog/Light, Chromatic Aberration, TAA, Motion Blur, and Digital Noise.
HUD is minimal, transparent glass, showing only contextual system health (GPU, inference progress, confidence).

## TECHNOLOGY STACK & PERFORMANCE
Next.js 15, React 19, R3F, Three.js, WebGPU, WGSL/GLSL, Motion One. 
Target: 144 FPS desktop, WebGPU first. Memory < 3GB VRAM via dynamic LOD and occlusion culling.

## ARCHITECTURAL PILLARS (The Bibles)
Future implementation will be divided into specific modules:
1. **UX Specification** (Interaction flows)
2. **3D World Specification** (Scene, physics, materials, lighting)
3. **AI Cognition Visualization** (Memory and reasoning states)
4. **Animation Bible** (Timings, transitions)
5. **Shader Bible** (Custom GLSL/WGSL for the Core and particles)
6. **Interaction Bible** (Hover states, volumetric shockwaves)
7. **Performance Bible** (WebGPU optimization, memory management)
