# 🦁 PREDATOR V45 - ШВИДКИЙ СТАРТ

## 🎯 Доступ до інтерфейсу

### Варіант 1: Через тунель (РЕКОМЕНДОВАНО)
Якщо сервер запущений і тунель активний:

```bash
# Запустити тунель до сервера
./scripts/server-tunnel.sh start

# Відкрити браузер
open http://localhost:9080
```

**Порт 9080** = Основний веб-інтерфейс v45 (через тунель до сервера)

### Варіант 2: Локальний запуск (якщо тунель недоступний)
Якщо потрібен локальний UI (може не працювати через обмеження системи):

```bash
./V45_GOLDEN_START.sh
```

Відкрити: `http://localhost:3030`

## 🔌 Перевірка підключень

### Бекенд (API)
```bash
curl http://localhost:3000/health
```

Має повернути: `{"status":"healthy"}`

### Фронтенд
Відкрийте в браузері:
- **http://localhost:9080** - Основний UI (тунель до сервера)
- **http://localhost:3000** - Бекенд API
- **http://localhost:3001** - Grafana моніторинг

## 🚀 Оновлення v45 на сервері

Якщо ви змінили код локально і хочете оновити версію на сервері:

```bash
./scripts/update-frontend.sh
```

Скрипт:
1. Синхронізує файли з `apps/predator-analytics-ui/` на сервер
2. Перезбудує Docker-контейнер фронтенду
3. Перезапустить сервіс

## ⚙️ Налаштування

### Порти:
- **3000** - Backend API (локальний)
- **3030** - Frontend UI v45 (локальний, якщо запущено)
- **9080** - Frontend UI v45 (тунель до сервера)
- **9090** - Backend API (тунель до сервера)

### Конфігурація API:
Фронтенд автоматично підключається до бекенду на `localhost:3000`.

Якщо потрібно змінити:
- Файл: `apps/predator-analytics-ui/src/services/api.ts`
- Змінна: `API_BASE_URL`

### Login Bypass:
У v45 увімкнено автоматичний вхід без пароля.

Якщо потрібно змінити:
- Файл: `apps/predator-analytics-ui/src/App.tsx`
- Рядок 31: `const [appState, setAppState] = useState<'BOOTING' | 'LOGIN' | 'READY'>('READY');`

## 🐛 Вирішення проблем

### UI показує "SYSTEM OFFLINE"
1. Перевірте, що бекенд запущений: `curl http://localhost:3000/health`
2. Перевірте тунель: `./scripts/server-tunnel.sh status`
3. Перезапустіть тунель: `./scripts/server-tunnel.sh restart`

### Порт зайнятий
```bash
# Звільнити порт 3030
lsof -ti :3030 | xargs kill -9

# Або порт 9080
lsof -ti :9080 | xargs kill -9
```

### "Operation not permitted" при запуску
Це обмеження macOS на папку проекту. Використайте тунель до сервера замість локального запуску.
